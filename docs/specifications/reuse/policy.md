# Reuse Policy

Document
- ID: reuse-policy
- Version: 1.0
- Status: Stable

Purpose
- Define adoption rules
- Define ownership boundaries
- Prevent unnecessary implementation

Rules

Knowledge

MUST
- Adopt proven knowledge
- Reference authoritative sources

MUST NOT
- Rediscover established knowledge

Specification

MUST
- Adopt open specifications before defining new specifications
- Extend adopted specifications only when required

MUST NOT
- Redefine adopted specifications
- Fork adopted specifications

Architecture

MUST
- Adopt proven architecture patterns
- Extend architecture patterns only for differentiated capability

MUST NOT
- Reinvent established architecture patterns

Algorithm

MUST
- Adopt proven algorithms
- Extend algorithms only for differentiated capability

MUST NOT
- Reinvent established algorithms

Implementation

MUST
- Adopt mature open source implementations
- Build adapters instead of forks
- Contribute improvements upstream when practical

MUST NOT
- Fork mature implementations without architectural justification
- Reimplement commodity software

Ownership

Sovrunn MUST own
- Semantic model
- Semantic planning
- Semantic execution
- Cross engine interoperability
- Unified capability model
- Autonomous intelligence

Sovrunn MUST NOT own
- Commodity standards
- Commodity protocols
- Commodity algorithms
- Commodity infrastructure
- Commodity runtimes

Decision Order

1 Adopt knowledge

2 Adopt specification

3 Adopt architecture pattern

4 Adopt algorithm

5 Adopt implementation

6 Extend

7 Build

8 Own

Exception

Deviation from this policy

MUST
- Demonstrate differentiated capability
- Demonstrate measurable benefit
- Be approved through RFC
