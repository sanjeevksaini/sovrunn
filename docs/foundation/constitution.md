# Constitution

Document
- ID: constitution
- Version: 1.0
- Status: Stable

Purpose
- Define immutable platform laws

Applies To
- Foundation
- Specifications
- Architecture
- Implementation
- RFC

Laws

Semantics

MUST
- Preserve semantic intent
- Separate semantics from implementation
- Separate semantics from execution

MUST NOT
- Couple semantics to protocol
- Couple semantics to engine

Interoperability

MUST
- Support heterogeneous systems
- Preserve protocol independence
- Preserve engine independence

Architecture

MUST
- Separate concerns
- Prefer composition
- Support independent evolution

MUST NOT
- Introduce cyclic dependencies

Ownership

MUST
- One concept one owner
- Reuse before define
- Reference before duplicate

MUST NOT
- Duplicate ownership
- Redefine adopted standards

Documentation

MUST
- AI first
- Human friendly
- Information dense
- Minimum token consumption
- One concept per document
- One decision per RFC
- Stable vocabulary
- Canonical abbreviations
- Markdown with structured fields

Evolution

MUST
- Preserve backward compatibility
- Prefer additive change
- Version specifications
- Require RFC for breaking change

Compliance

Every Specification

MUST
- Conform to this Constitution

Every Architecture

MUST
- Conform to Specifications

Every Implementation

MUST
- Conform to Architecture

Every RFC

MUST
- Preserve this Constitution
