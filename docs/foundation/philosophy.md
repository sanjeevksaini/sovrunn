# Philosophy

Document
- ID: philosophy
- Version: 1.0
- Status: Stable

Purpose
- Define enduring engineering philosophy
- Guide platform decisions
- Guide specifications
- Guide architecture
- Guide implementation
- Guide documentation

Beliefs

Semantics
- Meaning over syntax
- Intent over execution
- Semantics over implementation

Platform
- Platform over engine
- Interoperability over isolation
- Portability over lock-in
- Composition over duplication
- Open standards over proprietary design

Architecture
- Specification before implementation
- Architecture before optimization
- Stable contracts before features
- Separation before integration
- Determinism over ambiguity
- Additive evolution before replacement

Ownership
- Reuse before build
- Reuse before define
- Reference before duplicate
- Own only differentiated capability
- Single source of truth

Runtime
- Stateless before stateful
- Horizontal scale before vertical scale
- Push down before execute locally
- Fail fast
- Recover by replacement

Performance
- Minimize latency
- Minimize copies
- Minimize serialization
- Minimize allocations
- Measure before optimize

Documentation
- AI first
- Human friendly
- Information dense
- Minimum token consumption
- Stable vocabulary
- Canonical abbreviations
- One concept per document
- One decision per RFC
- Markdown with structured fields

Success
- Simplicity
- Predictability
- Portability
- Extensibility
- Interoperability
- Longevity
