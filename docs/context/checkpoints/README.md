# Architecture Checkpoints

This directory stores compact summaries after major architecture sessions, feature completions, monthly reviews, and phase gates.

Checkpoint files are not automatically authoritative. They become authoritative only after accepted decisions and baseline files are updated.

## Naming

Use:

```text
YYYY-MM-DD-topic-checkpoint.md
```

## Required Contents

- decisions confirmed
- decisions changed
- new open questions
- rejected ideas
- docs to update
- impacted features
- next recommended action
